# Usage Example: Modified Pluto with JSON Input

## Overview

The modified Pluto program now reads JSON emotion analysis data (generated by LLMs using the plutchik_analysis_prompt.md) and creates Plutchik emotion wheel visualizations.

## Key Changes

1. **Input Format**: Changed from text files to JSON files
2. **Analysis Method**: Reads pre-analyzed emotion data instead of performing text analysis
3. **3D Visualization**: Uses mild/moderate/intense intensity levels for richer visualization
4. **Metadata Support**: Displays analysis metadata like dominant emotions and confidence scores

## JSON Structure Required

The program specifically looks for the `part2_intensity_levels` section:

```json
{
  "analysis_metadata": {
    "dominant_emotions": ["anticipation", "fear", "trust"],
    "overall_emotional_tone": "Tense anticipation mixed with experienced confidence",
    "confidence_score": 0.88
  },
  "part2_intensity_levels": {
    "joy": { "mild": 0.1, "moderate": 0.0, "intense": 0.0 },
    "trust": { "mild": 0.2, "moderate": 0.4, "intense": 0.0 },
    "fear": { "mild": 0.2, "moderate": 0.4, "intense": 0.1 },
    "surprise": { "mild": 0.2, "moderate": 0.0, "intense": 0.0 },
    "sadness": { "mild": 0.1, "moderate": 0.0, "intense": 0.0 },
    "disgust": { "mild": 0.0, "moderate": 0.0, "intense": 0.0 },
    "anger": { "mild": 0.1, "moderate": 0.0, "intense": 0.0 },
    "anticipation": { "mild": 0.3, "moderate": 0.4, "intense": 0.1 }
  }
}
```

## How It Works

1. **JSON Parsing**: The `JSONEmotionParser` class reads and validates the JSON structure
2. **Data Conversion**: Converts the intensity levels to the format expected by pyplutchik:
   ```python
   emotions_degrees = {
       'joy': [0.1, 0.0, 0.0],      # [mild, moderate, intense]
       'trust': [0.2, 0.4, 0.0],
       'fear': [0.2, 0.4, 0.1],
       # ... etc
   }
   ```
3. **Visualization**: Calls `plutchik(emotions_degrees)` to generate the 3D emotion wheel

## Example Usage

```bash
# Basic usage
./pluto sample_analysis.json

# With verbose output
./pluto sample_analysis.json --verbose

# Custom output path
./pluto analysis.json --output my_emotion_wheel.png

# Custom title
./pluto analysis.json --title "Story Emotion Analysis"
```

## Workflow Integration

1. **Text Analysis**: Use an LLM with the `plutchik_analysis_prompt.md` to analyze text and generate JSON
2. **Visualization**: Use the modified Pluto tool to create the emotion wheel from the JSON
3. **Results**: Get a beautiful Plutchik wheel showing 3-dimensional emotion intensities

This approach separates the complex emotion analysis (done by LLMs) from the visualization (done by the specialized pyplutchik library), making the system more modular and powerful.
